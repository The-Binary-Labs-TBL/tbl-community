# Contributing to `tbl-community`

Thank you for helping shape the community space for The Binary Labs.

This repository is different from a product repository: its primary purpose is conversation, coordination, and shared guidance.

## What contributions are welcome

- improvements to the community `README`
- clearer contribution and moderation guidance
- updates that help contributors navigate discussions
- small repository improvements that make collaboration easier

## Before opening a change

- Keep edits focused and easy to review
- Prefer clarity over policy-heavy wording
- If the change affects contributor expectations, explain why in the pull request

## Discussion-first workflow

For broader community changes, start with a discussion before opening a pull request. This is especially helpful for:

- moderation or governance changes
- major wording updates that affect community tone
- new repository conventions

## Pull request expectations

Please include:

- a short summary of the change
- the reason for it
- any affected community workflow or guidance
- screenshots if the change affects rendered content in a visible way

## Communication

Be respectful, stay specific, and assume good intent. When you disagree, focus on the proposal or behavior rather than the person.

For behavior expectations, see [`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT.md).
